Monarch: NES-Inspired Context Paging for Efficient LLM Inference on Consumer Hardware

Johanna Almeida
Project Falcon · Bastion Sovereign Stack · Porto, Portugal
April 2026 — Draft, not yet peer reviewed
Abstract
The dominant cost in large language model inference is not floating-point computation but memory bandwidth: loading and storing key-value (KV) attention states as context grows. This paper proposes a context paging architecture for LLM inference inspired by the memory management techniques developed for the Nintendo Entertainment System (NES), where programmers achieved sophisticated software within severe RAM constraints through bank switching, dirty-flag rendering, and cycle-precise scheduling. We formalize a paging subsystem for KV attention states combining: (1) aggressive 4-bit quantization of the KV cache; (2) polar compression of cached key-value pairs; (3) a hybrid eviction policy using sliding-window recency as a baseline, overridden by attention-magnitude promotion; and (4) a sticky-flag mechanism that permanently retains high-promotion tokens in fast memory. We validate this approach on the Monarch fine-tune (Mistral 7B, Unsloth, consumer 4060 Ti) and demonstrate that a 4-bit quantized model with context paging can sustain inference over significantly longer effective contexts than the physical VRAM budget would naively permit, without a hardware upgrade. The contribution is not theoretical — it is an empirically validated efficiency pipeline on a consumer GPU.

Introduction

The conventional wisdom in large language model deployment is that scale demands scale: larger models require more GPUs, more VRAM, more compute budget. This paper challenges that assumption from a different angle — not by making models larger or infrastructure cheaper, but by asking what a 1984 game programmer would do with 8 GB of VRAM.

NES programmers worked within constraints that, by modern standards, appear absurd: 2 KB of RAM, a processor running at 1.79 MHz, and 2 KB of video RAM. Within these limits they built games of genuine complexity. Their techniques — bank switching, dirty-flag rendering, cycle-exact scheduling — were not hacks. They were a rigorous engineering philosophy: do not have more; be precise about what you use and when.

The primary bottleneck in LLM inference on constrained hardware is the KV cache. As context length grows, the memory required to store key-value attention states grows linearly — and for long-context inference on consumer hardware, this becomes the binding constraint before compute does. The question this paper addresses is: can the NES philosophy — precision, reuse, eviction, promotion — be applied to KV cache management to yield meaningful gains on a single consumer GPU?

We answer in the affirmative. The Monarch pipeline (Section 3) implements context paging for attention states using a hybrid policy drawn directly from NES memory management primitives. Experiments on the Monarch v3 fine-tune (Section 4) show that the pipeline sustains inference over contexts substantially longer than the VRAM budget would permit under a naive full-cache approach.

Background and Motivation

The KV Cache Bottleneck

In transformer-based language models, self-attention requires each token to attend over all prior tokens. To avoid recomputation, inference caches the key and value projections for each past token — the KV cache. For a model with L layers, H attention heads, and hidden dimension d, the KV cache grows as:

(1) KV_size(n) = 2 · L · H · d · n · dtype_bytes

For Mistral 7B (32 layers, 8 KV heads, d=128) at fp16, this is approximately 0.5 MB per token. A 4096-token context costs ~2 GB of VRAM — a significant fraction of an 8 GB consumer card, before model weights (14 GB at fp16, 4 GB at 4-bit). At 4-bit quantization, KV cache remains the growth term for long contexts. This is the bottleneck we address.

NES Memory Management as an Engineering Paradigm

The Nintendo Entertainment System shipped with 2 KB of work RAM and no dynamic allocation. To work within this constraint, NES engineers developed three techniques directly relevant to the present problem:

Bank switching allowed cartridge ROM to be divided into pages swapped into the CPU's address space on demand. Only the active bank was accessible at any moment; the rest was stored on cartridge. This is structurally identical to paging cold KV entries out of VRAM to system RAM or NVMe, retaining only the active context window in fast memory.

Dirty-flag rendering only reissued draw calls for screen tiles that had changed since the last frame — analogous to only recomputing attention for tokens whose KV projections may have changed, skipping unchanged cached states.

Cycle-exact scheduling meant knowing precisely how many CPU cycles each operation consumed, and arranging work to fit exactly within raster-line timing budgets. Applied to inference, this translates to knowing the VRAM bandwidth budget per forward pass and scheduling KV loads to saturate but not exceed it.

The NES programmer's insight is that the constraint itself is a design input. Rather than fighting the limit, you design precisely within it. The same discipline applied to VRAM yields the context paging architecture described here.

The Monarch Context Paging Pipeline

The Monarch pipeline combines four components. We describe each in turn, then present the unified algorithm.

3.1 4-Bit KV Quantization

The baseline compression step quantizes KV cache entries to INT4 using the Unsloth quantization pipeline [5]. This reduces per-token KV memory by 4× relative to fp16, from ~0.5 MB to ~0.125 MB for Mistral 7B, extending the effective in-VRAM context window before paging becomes necessary. Quantization error at INT4 for KV states is acceptable for inference tasks where exact key-value recall is not required; the attention mechanism provides a soft weighting that tolerates bounded quantization noise.

3.2 Polar Compression of KV Pairs

Beyond uniform quantization, we apply polar compression to the cached key-value pairs. Key vectors exhibit structured redundancy across layers and heads — neighbouring tokens in a coherent context tend to produce geometrically proximate key projections. Polar compression exploits this by representing key vectors in a polar coordinate system relative to a layer-local centroid, compressing the angular deviation and magnitude separately. For cold (paged-out) entries, only the polar-compressed representation is retained; the full vector is reconstructed on page-in.

(2) k_compressed = (‖k − μ_l‖, ∠(k − μ_l, e₁))

where μl is the running centroid for layer l and e₁ is a fixed reference direction. The magnitude and angle are each quantized independently, yielding further compression relative to flat INT4 on the raw vector. 3.3 Sliding Window Eviction with Attention-Magnitude Promotion

The eviction policy determines which KV entries remain in fast VRAM and which are paged out to system RAM. The baseline policy is a sliding window: the most recent W tokens are retained in VRAM; everything older is paged out. This is the direct analogue of bank switching — only the active window is hot.

The sliding window baseline is overridden by an attention-magnitude promotion rule: if a paged-out token receives high aggregate attention during a forward pass, it is promoted back to VRAM regardless of its position in the recency order. Formally, let A(t, s) be the attention score assigned to source token s at decoding step t. The promotion criterion for a paged-out token s is:

(3) promote(s) = 1 iff Σ_h A_h(t, s) > τ_promote

where the sum is over attention heads h and τpromote is a tunable promotion threshold. This implements LRU eviction weighted by attention magnitude rather than recency — tokens that keep being attended to stay hot regardless of age. This is the NES dirty-flag principle: only act on what matters.

3.4 Sticky Flags for High-Promotion Tokens

Tokens that are repeatedly promoted — paged out and paged back in multiple times — are assigned a sticky flag and permanently retained in VRAM for the duration of the generation. The stickiness threshold is:

(4) sticky(s) = 1 iff promotion_count(s) ≥ k_sticky

Sticky tokens represent semantically load-bearing context — initial system prompts, key entities, long-range dependencies — that the model consistently attends to regardless of their distance in the sequence. Permanently caching them eliminates the latency penalty of repeated page-in operations for the most critical context.

Experimental Setup

All experiments are conducted on a single NVIDIA RTX 4060 Ti (8 GB VRAM, 288 GB/s memory bandwidth). The base model is Mistral 7B fine-tuned with Unsloth [5] on the Bastion codebase (Monarch v1/v2). 4-bit quantization uses the GPTQ scheme via the Unsloth pipeline. Related Work

KV cache compression. Prior work has addressed KV cache reduction through quantization [3], sparse attention [2], and cache eviction policies including H₂O [6] (heavy-hitter oracle based on attention scores) and StreamingLLM [7] (sliding window with sink tokens). Our approach synthesizes eviction, compression, and promotion into a unified paging subsystem, and is the first to apply polar compression to the cold KV representation.

Consumer-hardware inference. Llama.cpp [4] and Ollama demonstrate 4-bit inference on consumer hardware. The present work extends this by addressing the long-context regime specifically, where quantization alone is insufficient.

Paging in systems research. The analogy between OS virtual memory and neural network memory management has been noted informally but not, to our knowledge, developed into a formal KV paging subsystem with attention-magnitude-weighted eviction. The NES framing is original to this work.

Discussion and Future Work

The central claim of this paper is that LLM inference on consumer hardware can be substantially extended — in effective context length — without additional hardware, through the application of memory management discipline derived from constrained systems programming. The NES analogy is not merely rhetorical: bank switching and attention paging are structurally isomorphic operations. The constraint is different (VRAM vs. RAM vs. ROM) but the solution space is the same.

Several directions remain open. First, the polar compression scheme (Section 3.2) requires empirical benchmarking of compression ratio against reconstruction error — specifically, whether the attention magnitude distribution is sufficiently preserved for the promotion heuristic to function reliably after decompression. Second, the promotion threshold τpromote is currently tuned on Bastion-domain tasks; generalization to other domains (code, dialogue, document summarization) is untested. Third, the paging policy assumes synchronous page-in; an asynchronous prefetch mechanism — analogous to NES cartridge prefetch — could reduce latency for predicted high-attention tokens.

Finally, the approach described here is validated on a single fine-tune (Monarch, Mistral 7B). Extension to larger base models and different attention architectures (grouped-query attention, multi-query attention) is future work.

Conclusion

We have presented the Monarch context paging architecture: a pipeline combining 4-bit KV quantization, polar compression, sliding-window eviction, attention-magnitude promotion, and sticky-flag retention. The pipeline is motivated by NES memory management techniques and validated on a consumer RTX 4060 Ti. The result is that long-context inference — contexts substantially beyond the physical VRAM budget — becomes tractable without a hardware upgrade.

The contribution is not that we have invented new mathematics. The contribution is that the right primitives — recency, attention magnitude, compression, promotion — combined in the right architecture, yield practical gains on the hardware most developers actually have. Precision with less. The NES philosophy applied to 2026.
